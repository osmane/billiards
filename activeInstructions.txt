Thank you for the updates. The UI indicator for the strike point is now working perfectly, which confirms that part of the preset data is being processed correctly.

However, we are still facing the core issue: the executed shots do not curve, and there is no visual change to the cue stick's elevation angle when a preset is selected. We have analyzed the difference between the successful physics tests and the live UI, and we believe we have identified the root cause.

Problem Analysis:

The successful physics tests work in an isolated environment where shot parameters are set once and executed immediately. The game UI, however, is dynamic.

We believe that after a piqué preset is selected, the game's main aiming system (which tracks mouse and keyboard inputs for normal aiming) continues to run in the background. This system appears to be overwriting or resetting the cue's vertical elevation angle back to zero before the "Hit" button is pressed.

As a result, by the time the shot is executed, the physics engine receives the correct strike point but an elevation angle of 0°, leading to a straight shot with no curve. The fact that the 3D cue model never visually tilts is a symptom of this same issue—the elevation value is being lost.

Requested Solution:

To resolve this, we need to create a "preset lock" mode. When a user selects a piqué preset from the dropdown:

The system should "lock in" all parameters from that preset, especially the vertical elevation angle.

While a preset is active, the standard aiming controls (like mouse movement for aiming or manual elevation changes) should be temporarily disabled to prevent them from resetting the locked-in preset values.

The 3D model of the cue stick must be visually updated to reflect the locked-in elevation angle, providing clear feedback to the user.

Selecting the "None" or default option in the dropdown should disable the lock and return the system to normal aiming mode.

This will ensure that the exact parameters from the successful tests are passed to the physics engine, allowing us to see the expected curves in the UI.
